# Concept Lifecycle — ACM

**Status:** Canonical for M3  
**Companions:** [`CONCEPT_ARCHITECTURE.md`](CONCEPT_ARCHITECTURE.md) · [`EXPERIENCE_MODEL.md`](EXPERIENCE_MODEL.md)

## Stages

```
birth (nucleus)
  → growth
  → stable
  → mature
  ⇄ dormant
  → retired
  ↑ resurrection (via new evidence)
```

| Stage | Meaning |
|-------|---------|
| **Nucleus** | Emerging; few Experiences; provisional |
| **Growing** | Repeated support; still forming |
| **Stable** | Reliable meaning under default recognition |
| **Mature** | High strength/confidence; abstraction-ready |
| **Dormant** | Weakened / unused; retained |
| **Retired** | Out of default circulation; retained for history |

Plus structural evolutions: **abstraction**, **specialization**, **merge proposal**, **split candidate**.

## Birth

Triggered by Experience absorption when no adequate Concept matches. Identity schema Concepts remain privileged nuclei (M1) but still sit in the Concept store.

## Growth / strengthening / weakening

- Congruent Experience → strengthen  
- Neglect / soft decay → weaken toward dormant  
- Attribute contradiction → confidence pressure; attribute versioning  

## Merge / split

Recorded as proposals/candidates in M3. Silent high-impact merges of mature Concepts are forbidden (Policy/assent direction, aligned with Identity and future self-improvement).

## Interaction with Identity & Experiences

| Organ | Interaction |
|-------|-------------|
| Experiences | Sole evidentiary source for ordinary Concepts |
| Identity | Privileged Concepts; experience still grounds content |
| Future Associations | Concepts are primary endpoints — see [`ASSOCIATION_MODEL.md`](ASSOCIATION_MODEL.md) |
| Future Remembering | Should prefer Concept + Association activation over raw Experience scans |

## Long-term architectural direction

```
Experience → Concept → Association → Remembering → Reflection → Learning → Prediction → Planning
```

Future organs should avoid manipulating raw Experiences directly when Concepts suffice. This is direction, not an M3 implementation mandate.

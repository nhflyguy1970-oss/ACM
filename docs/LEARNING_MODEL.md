## Definition

Learning applies **auditable Adaptation Records** to living cognitive structures (Concepts, Associations, preference confidence, limited identity confidence, goal importance) so future Remembering changes — without rewriting Experiences.

## Lifecycle

Reflection → Reflective Experience → Adaptation candidate → automatic apply **or** proposal → optional Offline consolidation → query via `what_have_i_learned`.

## Adaptation

Each Adaptation cites reflective triggers, target ids, before/after vectors, and governance class (`automatic` | `proposed` | `assented` | `rejected` | `rolled_back`).

## Generalization & transfer

Repeated congruent Reflective `sufficient`/`consistency`/`pattern` outcomes strengthen shared Concept/Association neighborhoods. Transfer is neighborhood-limited via Activation reuse — not invented links.

## Confidence

Reflection evaluates. Learning adjusts confidence of Concepts/attributes within caps. Offline Cognition stabilizes through replay batches.

## Identity evolution

Low-impact: slight confidence nudges on identity Concepts from congruent self-reflective evidence. High-impact adopt/supersede remain **assent-only** (never automatic Learning).

## Ownership

Learning owns durable adaptation. It never assumes Reflection evaluation, Experience history, or architectural self-modification.

## Ownership

| Organ | Question | Exclusive |
|-------|----------|-----------|
| Attention & Memory Priority (M9) | What deserves attention and continued investment? | Allocation + priority investment |
| Accessibility & Forgetting (M10) | What should become harder to remember? | Accessibility stages / cool / reactivate |

They cooperate: high priority slows cool; reactivation may invite renewed Attention investment. They never swap jobs.

## Lifecycle

```
Encode (Attention gates)
  → Living Priority investment
  → Remembering / Reflection / Learning (modulated by Attention + Accessibility)
  → Offline replay (priority-ranked)
  → Forgetting cools neglected / weak paths
  → Strong cue / identity / Learning → Reactivation
  → Future Remembering
```

## Interactions

| From → To | Effect |
|-----------|--------|
| Attention → Activation | Energy scale for cues |
| Priority → Offline | Replay ranking of hot concepts |
| Priority → Forgetting | Cool resistance |
| Forgetting → Activation | Accessibility multiplier + dormancy |
| Remembering / Learning → both | Investment + accessibility recovery |
| Offline → Forgetting | Requests weak-association cool |

## Architecture rules

- Singular Activation Architecture
- Attention ≠ planning
- Priority ≠ decision making
- Forgetting ≠ deletion
- Accessibility ≠ history loss

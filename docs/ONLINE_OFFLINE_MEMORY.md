## Online cognition (world-facing & evaluative)

| Organ | Question |
|-------|----------|
| Encode / Experience | What happened? |
| Concept / Association | What is this? / How related? |
| Remembering | What do I remember? |
| Reflection | What do I think about what I remember? |
| Learning | What have I learned? (applies adaptations online from Reflection) |

Online may interact with host encode/remember/reflect/learn verbs.

## Offline cognition (memory organization)

| Organ | Question |
|-------|----------|
| Offline Cognition (Sleep) | What should become long-term memory? |

Offline **never** opens sensors, never invents Experiences, never talks to hosts for new content. It reorganizes: replay, consolidate, propose, cool weak links.

## Ownership boundary

| Concern | Online Learning | Offline Cognition |
|---------|-----------------|-------------------|
| Adaptation from one Reflective Experience | Yes | May batch-replay many |
| Fabricate history | Never | Never |
| High-impact merge | Propose / assent | Propose only |
| Weak edge dormancy | Rare | Yes (consolidation cool) |
| Architecture change | Never | Never |

## Shared lifecycle

```
Online: Experience → Remember → Reflect → Learn (Adaptation)
Offline: Replay Reflective/high-access → Consolidate Adaptations → Organize
Future Remembering benefits from both
```

## Future Forgetting

Depends on both: Learning sets durable lessons; Offline cools unused accessibility; Forgetting will own explicit accessibility policy (not implemented).

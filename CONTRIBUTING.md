# Contributing

Thank you for helping build ACM.

## Before you change code

1. Read `docs/MEMORY_DESIGN_PRINCIPLES.md`.
2. Check that the change advances practical cognition, not only storage/search convenience.
3. Prefer emergent mechanisms over hard-coded scripts for identity, confidence, and preference.

## Process

1. Open an issue describing the cognitive problem or measured regression.
2. Keep PRs milestone-scoped.
3. Include tests (unit + behavioral or cognitive as appropriate).
4. Update docs / DECISION_LOG if architecture or public API shifts.
5. Do not add Aria / Mission Control / AI-Platform imports.

## Style

- Python 3.11+
- `ruff check` clean
- Small commits with clear cognitive intent
